Instance

Instance stores presets for each individual staking, and own treasury accounts for rewards,

Configure

Move to Instance tab on navbar and click Configure
  • Collection: put your collection address, if you don’t have it - put any NFT mint address
  • Specification: Engine supports two variants of staking: Stake is a standard, holders get unit reward for every unit time of staking, can unstake or claim intermediate rewards at any time. Lock provides a single reward for a unit time and holders can not unstake until the end of period.
  • Unit Time is a period for reward accrual.
  • Reward Tokens: award your holders with combination of Utility Tokens, Solana or Stable Coins, Engine supports up to 5 tokens at the same time. For an utility token put mint address.

    • Deploy

    Click Configure and check setting before deploying
    You want be able to reset Specification or Reward Token list after you deploy or add new NFT’s mints

    States

    After deploying an Instance you would recieve NFT that represents current state
    Init: you can make any changes, but holders can not stake yet
    Running: holders can stake and unstake, but you cannot make any changes
    Pause: you can make change until time and reward values, but holders cannot stake until you launch it again, however anyone can unstake their NFTs anytime

    Retrieve

    If you have lost your repository you can retrieve Instance from Instance Engine NFT, at the start in Instance tab you would see very Instances owned by your account, choose one to load.
    You would restore full access only if you have already launched Instance

    Update Time

    After very time you launch the Instance the update time would be update Engine offers honest rules for both NFT holder and project owner.
    Rewards for staking calculates as the sum of rewards based on unit time and values before last update time and rewards based on new values for remaining time, in other words, changes would be applied for periods after update time.
