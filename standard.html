Standard

Metaplex V2

In Solana ecosystem conventional standard is introduced by metaplex foundation: https://docs.metaplex.com/, it shares the same idea with ERC20 and ERC721 in ETH, providing Metadata and Collection accounts common for NFT recognition by subsystems as wallets and marketplaces. Engine is supposed to work with it and reveals the potential only with the latest Metaplex V2 Metadata standard: https://docs.metaplex.com/programs/token-metadata/token-standard. If you are using Magic Eden or Candy Machine V2 for minting, don’t worry, all of them using latest versions.

Hashes

Every NFT in Solana is represneted as signle token, in other words every NFT hash it’s own mint(hash). Look for mint address on Solscan:

Hash List

Hash List is a complete list of all mints from your collection. Engine requires it to strictly identify witch NFT belongs to staking. If you are using Candy Machine on mainnet you can retrieve hash list from: https://magiceden.io/mintlist-tool, otherwise try from Collection address with our free code: https://github.com/ZarveTestnet/demo_collection

Collection

Collection account represents your collection ownership, all your NFTs would keep reference to it and must be verified. That is an indicator for marketplaces that NFTs are from the same collection and made by the same author. Look for collection on Solscan:
